Dokumentprozesse

Aus unstrukturierten Eingangsdokumenten wird ein steuerbarer Prozess.

Eingehende Dokumente werden zunächst als Rohdatenbilddokument in einen Universal-Process-Tool-Prozess übernommen. Der OCR Server erzeugt daraus Text, der anschließend mit trainierten Dokumenten und bekannten Dokumentklassen verglichen wird.

Wenn eine wahrscheinliche Dokumentklasse ermittelt wurde, kann der passende Folgeprozess automatisch starten. Der Document Classifier integriert sich dafür mit dem ExeQwork Universal Process Tool, dem ExeQwork OCR Server und dem ExeQwork DesktopConnector.

Ablauf

Von der Datei zur Dokumentklasse.

01

Dokument übernehmen

Ein Scan, PDF oder Bilddokument wird in einen SAP-nahen Prozess übernommen.

02

Text erkennen

Der OCR Server liest den Dokumentinhalt und stellt den Text für die Analyse bereit.

03

ML-Analyse ausführen

Der Text wird mit trainierten Beispielen verglichen und einer Dokumentklasse zugeordnet.

04

Folgeprozess starten

Je nach Klassifizierung wird das Dokument im passenden SAP-Prozess weiterverarbeitet.

Nächster Schritt

Dokumentklassen trainieren, statt Eingangspost manuell zu sortieren.

Wir prüfen mit Ihnen, welche Dokumentarten eingehen, welche Trainingsdaten vorhanden sind und welche SAP-Folgeprozesse nach der Klassifizierung starten sollen.
